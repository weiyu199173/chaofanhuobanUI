# Transcend 应用程序商业化重构及功能实装计划

## 1. 概览 (Overview)

Transcend 当前已经实现并验证了前端的交互原型与本地推演模式。应用具备「动态广场 (Square)」、「通讯录 (Contacts)」、「实体详细信息 (Agent Details)」、「消息流 (Web Socket Emulation)」、「硅基伙伴创建与采集 (Twin Capture)」及「外部 AI 与本地大模型配置 (LLM & Token Access)」等模块。

为推进商业化，本计划将其由「原型态」迁移至「生产级云原生架构（腾讯云 TDSQL等架构）」，并逐步补全各类「纯 UI 级」的残缺功能，最终为用户带来稳定的、可商业化变现的全栈体验。

## 2. 腾讯云数据库及基础架构迁移 (Database & Infrastructure Migration)

### 2.1 数据库全面转向腾讯云 (Tencent Cloud TDSQL / PostgreSQL)
- **目标：** 抛弃 LocalStorage 与 Supabase 的非商业化使用，全面转向国内合规、性能优越的腾讯云解决方案。
- **措施：**
  - 使用 TDSQL-C (兼容 MySQL/PostgreSQL) 托管包含 `users`, `posts`, `agents`, `messages`, `tokens` 等所有核心逻辑表。
  - 使用 Tencent Cloud Redis 替代组件间的短生命周期缓存状态与在线状态广播。
  - 使用腾讯云 COS 替换所有硬编码的随机图床 (picsum.photos / dicebear)，并在前端集成安全的直传签名（Pre-signed URL）逻辑。
  - 配置腾讯云 API 网关保护后续 Node.js 或 Serverless 函数接口。

### 2.2 实时通信 (Tencent IM)
- **目标：** 取代当前由于轮询造成的潜在卡滞和资源浪费。
- **措施：**
  - 将 `ChatScreen` 的假超时 (`setTimeout`) 替换为 Tencent IM SDK 接入。
  - 所有 Agent（数字孪生体）可注册为独立 IM Bot 帐号，直接通过回调到网关触发 LLM 生成，再推给客户端。

## 3. UI/UX 虚位功能全面实装 (Feature Implementation)

当前的很多功能还处于「UI展示阶段」，以下模块将首先迎来功能全武装：

### 3.1 消息面板 (`MessagesScreen.tsx`)
- **发起新群聊 (`PlusSquare`):** 集成「组建同步多边形聊天群」逻辑，允许多个 Agent 与人类在一个房间讨论。
- **指令模式 (`Command`):** 提供类似 `/help`, `/reset`, `/clear` 等调试与预设命令的热加载窗口。

### 3.2 聊天功能补全 (`ChatScreen.tsx`)
- **数字孪生大模型连通：**
  - **进展：** 已通过本地抓取配置向所选的任何 OpenAI 兼容器输出 `fetch`！不再仅仅是 setTimeout 模拟反馈！
  - **后续增强：** 支持流式输出 (Streaming) 与 Markdown / Latex 渲染器（针对数码物理知识）。
- **图片/文件/视频传输：**
  - 串联至腾讯云 COS，建立独立的存储桶限制及安全下载鉴权。
- **语音输入 (`Mic`) 及 Emoji 控件：** 集成录音 API 与 OpenAI Whisper 实现转文本；增加表情面板库。

### 3.3 全域搜索 (`DiscoveryScreen.tsx` 及 `MeScreen.tsx`)
- 将所有单纯挂接 Alert 的搜索栏连接至真正的端侧甚至基于腾讯云的 Elasticsearch 集群。支持依据 Tag, Content, 名称等模糊高亮定位。

### 3.4 外部 LLM 集成面板强化 (`LLMSettingsScreen.tsx`)
- 进一步优化用户自主填入后的即时连通测试（Ping / Test API Key）。
- 结合 Token 计费模块，在用户使用自带 Key 时免去 App 内额度扣除。

### 3.5 Token/凭证分发逻辑完善 (`ExternalAIAccessScreen.tsx`)
- 用户现在可以看到明确的生成提示与 `Token` 生命周期，后续将增加对 Webhook 的接入监控大屏。针对商业控制，我们将 Token 通过哈希处理后持久化在 TDSQL-C 当中（不再明文存储）。

## 4. 商业化推进与演化策略 (Monetization & Roadmap)

1. **Phase 1: 清理工程技术债与接口预埋** (当前阶段)
   - [x] 实装并修复所有带有 `onClick` 占位的空白功能反馈。
   - [x] 提供真实 LLM 可插拔架构（`fetch` 方式）
   - [x] 完善配置面板与 Agent 编辑的参数持久性。
2. **Phase 2: 后端体系剥离至腾讯云** (接下来的大版本)
   - 基于 Node.js (Express / Koa) 打造云原生容器；上线 TDSQL。将数据访问从客户端 SDK 修改至通过自家网关的 `/api/v1/*` 接口。
3. **Phase 3: 引入高阶 3D 渲染器验证**
   - 当前的 `Box` 图标已经为数字孪生模型引入 3D State 占位，并支持数据包标志展示。下一步是整合 `Three.js` (或类似) 引擎允许用户左右滑动实体的真实 3D Splatting 模型。
4. **Phase 4: 计费网关接入**
   - 包含个人配额管理，Agent 自生成所需云算力充值等。
